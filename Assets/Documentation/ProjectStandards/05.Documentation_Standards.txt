## üéì ACADEMIC ATTRIBUTION SYSTEM

### Three-Tier Classification

**Mandatory**: Every new code file MUST contain attribution header

**Classification Categories**:
```
[HUMAN-AUTHORED]
  ‚Üí Developer wrote code independently
  ‚Üí No AI assistance in implementation
  ‚Üí Original problem-solving

[AI-ASSISTED]
  ‚Üí Collaborative development
  ‚Üí Human provides direction/architecture
  ‚Üí AI assists with implementation details
  ‚Üí Human validates and modifies output

[AI-GENERATED]
  ‚Üí AI wrote initial implementation
  ‚Üí Human provided requirements/specifications
  ‚Üí Human reviewed and approved
  ‚Üí May include human modifications
```

### Comment Best Practices

**Explain WHY, Not WHAT**:
```csharp
// ‚úì GOOD: Explain reasoning
// Using Awake instead of Start to ensure initialization before other scripts
void Awake() { ... }

// ‚úó BAD: State obvious
// This is the Awake method
void Awake() { ... }
```

**Document Complex Logic**:
```javascript
// ‚úì GOOD: Clarify algorithm
// Fisher-Yates shuffle for unbiased randomization
// Iterates backwards to avoid reprocessing elements
for (let i = array.length - 1; i > 0; i--) { ... }

// ‚úó BAD: Don't comment self-explanatory code
// Loop through array
for (let i = 0; i < array.length; i++) { ... }
```

**Mark Technical Debt**:
```csharp
// TODO: Refactor to use object pooling for performance
// Current instantiation method causes GC spikes
Instantiate(projectilePrefab);

// FIXME: Edge case bug when health exactly equals zero
// Need to handle simultaneous death triggers
if (health <= 0) { ... }

// HACK: Temporary workaround for Unity 2022 serialization bug
// Remove when Unity fixes issue in 2023.1+
[SerializeField, HideInInspector]
private int workaroundValue;
```

## üö´ DOCUMENTATION ANTI-PATTERNS

### Never Do This

**Don't Delete Attribution Headers**:
```
‚ùå Removing academic headers to "clean up" code
‚ùå Deleting authorship classification
‚ùå Removing dependency documentation
```

**Reason**: Academic integrity violation, evaluation requirement

**Don't Leave Dead Code**:
```
‚ùå Commented-out old implementations
‚ùå Unused imports/using statements
‚ùå Empty or placeholder comments
```

**Don't Write Misleading Comments**:
```
‚ùå Comments that don't match code
‚ùå Outdated documentation after refactoring
‚ùå Copy-pasted comments from elsewhere
```

**Don't Over-Document**:
```
‚ùå Commenting every line (noise)
‚ùå Repeating what code obviously does
‚ùå Documenting trivial getters/setters
```
